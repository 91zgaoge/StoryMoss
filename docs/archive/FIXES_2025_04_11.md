# StoryForge 修复记录 - 2025年4月11日

## 修复概述

本次修复解决了 StoryForge (草苔) v2.0 在 Windows 平台上运行时出现的两个主要问题：
1. **无法连接本地服务端口** - 前端无法连接到 Tauri 后端服务
2. **React 无限循环错误** - "Maximum update depth exceeded" 导致应用崩溃

---

## 问题 1：无法连接本地服务端口

### 问题描述
运行应用后，窗口显示"无法连接本地服务端口"，前端无法与 Tauri 后端通信。

### 根本原因
- Windows 系统上 `localhost` 可能解析为 IPv6 地址 `::1`
- Tauri 开发服务器绑定到 IPv4 的 `127.0.0.1`
- CSP (Content Security Policy) 配置不完整
- 权限配置使用了无效的权限标识符

### 修复详情

#### 1.1 Tauri 配置 (`src-tauri/tauri.conf.json`)
```json
{
  "build": {
    "devUrl": "http://127.0.0.1:5173"
  },
  "app": {
    "withGlobalTauri": true,
    "security": {
      "csp": "default-src 'self'; connect-src 'self' http://127.0.0.1:* http://localhost:* ws://127.0.0.1:* ws://localhost:*; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob: https:; font-src 'self' https:;"
    }
  }
}
```
**关键改动：**
- `devUrl` 从 `http://localhost:5173` 改为 `http://127.0.0.1:5173`
- 添加 `withGlobalTauri: true`
- 完善 CSP 策略，同时允许 `127.0.0.1` 和 `localhost`

#### 1.2 Vite 配置 (`src-frontend/vite.config.ts`)
```typescript
server: {
  port: 5173,
  strictPort: true,
  host: '127.0.0.1',
  hmr: {
    protocol: 'ws',
    host: '127.0.0.1',
    port: 5173,
  },
  cors: true,
}
```
**关键改动：**
- 明确设置 `host: '127.0.0.1'`
- 配置 HMR (热模块替换) 使用 WebSocket 协议
- 启用 CORS 支持

#### 1.3 权限配置 (`src-tauri/capabilities/main-capability.json`)
```json
{
  "permissions": [
    "core:default",
    "http:default",
    "http:allow-fetch",
    "http:allow-fetch-send",
    "http:allow-fetch-read-body"
  ]
}
```
**关键改动：**
- 移除无效的 `http:allow-request` 权限
- 添加有效的 HTTP 权限
- 添加更多窗口管理权限

#### 1.4 后端增强 (`src-tauri/src/lib.rs`)
- 添加 `health_check` 命令用于前端检测后端状态
- 改进数据库初始化错误处理
- WebSocket 服务器支持多端口尝试（8765-8769）

## 问题 2：React 无限循环错误

### 问题描述
应用启动后出现错误："Maximum update depth exceeded. This can happen when a component repeatedly calls setState inside componentWillUpdate or componentDidUpdate."

### 根本原因
- React Query 的重试机制与组件的 useEffect 形成循环
- `useStories` hook 在 Dashboard 组件中每次重试都触发状态更新
- 状态更新又触发重新渲染，形成无限循环

### 修复详情

#### 2.1 新建数据加载组件 (`src/components/DataLoader.tsx`)
```typescript
// 独立的数据加载组件，与渲染逻辑分离
export function DataLoader() {
  const hasAttemptedRef = useRef(false);
  
  // 先进行健康检查
  const { isSuccess: isHealthOk } = useQuery({
    queryKey: ['health'],
    queryFn: healthCheck,
    retry: 2,
    refetchOnWindowFocus: false,
  });

  // 健康检查通过后再加载数据
  const { data: stories } = useQuery({
    queryKey: ['stories'],
    queryFn: listStories,
    enabled: isHealthOk,  // 条件启用
    retry: 1,
    refetchOnWindowFocus: false,
  });

  // 使用 ref 确保只同步一次
  useEffect(() => {
    if (stories && !hasAttemptedRef.current) {
      hasAttemptedRef.current = true;
      setStories(stories);
    }
  }, [stories]);

  return null;
}
```
**关键改动：**
- 将数据加载逻辑从 UI 组件中分离
- 使用 `useRef` 确保数据只同步一次
- 先进行健康检查，通过后再加载故事数据

#### 2.2 简化 Dashboard 组件 (`src/pages/Dashboard.tsx`)
```typescript
export function Dashboard() {
  const stories = useAppStore((s) => s.stories);
  const hasHydrated = useRef(false);

  // 用户初始化只执行一次
  useEffect(() => {
    if (hasHydrated.current) return;
    hasHydrated.current = true;
    // ... 初始化逻辑
  }, [setCurrentUser]);

  // 不再直接调用 useStories
  // 从 store 获取数据
}
```
**关键改动：**
- 移除 `useStories` hook 调用
- 从 store 获取数据
- 使用 `useRef` 确保初始化只执行一次

#### 2.3 优化 React Query 配置 (`src/main.tsx`)
```typescript
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 1000 * 60 * 5,
      refetchOnWindowFocus: false,
      retry: 2,  // 限制重试次数
      retryDelay: (attemptIndex) => Math.min(1000 * 2 ** attemptIndex, 5000),
    },
    mutations: {
      retry: 0,  // 禁用 mutation 重试
    },
  },
});
```
**关键改动：**
- 限制查询重试次数为 2 次
- 限制重试延迟最大 5 秒
- 禁用窗口聚焦时自动刷新
